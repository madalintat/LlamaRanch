LlamaRanch

Documentation

A local AI runner and agent on top of it. Runs llama.cpp models on your own hardware behind one private OpenAI-compatible endpoint. Nothing leaves your machine.

01

Overview

LlamaRanch is two things behind one tray icon. A model server that runs GGUF models via llama.cpp and exposes them on a single private OpenAI-compatible endpoint. And a chat agent that routes each task to the right local model, swaps experts in and out to fit your memory, and calls tools when it needs them.

The endpoint lives at:

127.0.0.1:2276/v1 LOCAL

Point any OpenAI-compatible app at that URL and it works: Open WebUI, Continue, Zed, Cline, curl, any OpenAI SDK. macOS, Linux, Windows. Nothing leaves the valley.

02

Quick start: the wizard

One command covers the whole setup:

shell
$ npx @llamaranch/wizard

The wizard detects your hardware, installs llama.cpp with the right backend (Metal, CUDA, Vulkan, CPU), suggests and downloads models that fit your memory, writes your config, and installs the app. When it finishes, open the app. Everything is ready.

Need it headless, e.g. on a server? Run the router straight from the terminal:

shell
$ npx @llamaranch/wizard serve

The wizard is published on npm as @llamaranch/wizard and works on macOS, Linux, and Windows. If any step can't run automatically (no Homebrew, an unusual distro), it explains exactly what to do by hand.

03

Install by hand

Prefer a direct download? Grab the right installer for your OS from Releases.

OS Arch Formats
macOS Apple Silicon .dmg
Linux x86_64 · arm64 .deb · .AppImage · .rpm
Windows x86_64 · Arm .exe · .msi

You also need a llama-server binary from llama.cpp on your PATH:

macOS (Homebrew)
# macOS (Homebrew)
$ brew install llama.cpp

# Linux / Windows: grab a prebuilt from llama.cpp Releases
# https://github.com/ggml-org/llama.cpp/releases/latest
# (CPU, CUDA, Vulkan, Metal variants available)

First run finds llama-server on your PATH automatically. Open the tray popover, pick a model from the catalog (or drop in any .gguf), load it, and start chatting.

macOS builds are ad-hoc signed but not yet notarized. On the first launch, right-click the app and choose Open to bypass Gatekeeper.

04

Updating

The quickest path on any OS:

shell
$ npx @llamaranch/wizard update

The app also self-updates. LlamaRanch checks Releases on launch and shows a banner when a newer signed build is available.

In-place updates: .dmg (macOS), .exe and .msi (Windows), and .AppImage (Linux) all update in place. Click Update in the banner; the app downloads, checks the signature, and relaunches.

For Linux package formats, download the new file from Releases and run:

shell
# .deb
$ sudo dpkg -i LlamaRanch_*.deb

# .rpm
$ sudo rpm -Uvh LlamaRanch-*.rpm

Every update is checked against the signing key. A tampered build won't install. To stay on a specific version, download it directly from Releases.

05

The endpoint

127.0.0.1:2276/v1 is llama.cpp's llama-server, run and supervised by LlamaRanch in router mode. It handles multi-model load/unload, --fit memory sizing, jinja templates, and idle sleep. The API is OpenAI-compatible.

Key routes:

shell · list models
$ curl http://127.0.0.1:2276/v1/models
{ "data": [ { "id": "gemma-3-4b", "object": "model" } ] }
shell · chat completions 
$ curl http://127.0.0.1:2276/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"gemma-3-4b",
       "messages":[{"role":"user","content":"Hello"}]}'
shell · embeddings 
$ curl http://127.0.0.1:2276/v1/embeddings \
  -H "Content-Type: application/json" \
  -d '{"model":"nomic-embed-text","input":"The sky is blue"}'

Works with Open WebUI, Continue, Zed, Cline, curl, and any OpenAI SDK. Set the base URL to http://127.0.0.1:2276/v1 and leave the API key blank (or any string).

By default the server binds 127.0.0.1 only. To expose the endpoint to other devices on your network, set expose_to_network: true in your config. That binds 0.0.0.0. Off by default because the endpoint carries no authentication.

06

Models

LlamaRanch ships a catalog of 25+ curated GGUF models, one click to download. For gated repos on Hugging Face (such as Llama 3 family models), add your HF token in Settings. You can also drop any .gguf file into the models folder and it appears automatically.

Hardware fit via --fit: LlamaRanch sizes GPU layers and context to the memory you actually have. No manual tuning. Per-model context length and sampling are adjustable in the model detail view, with a live memory estimate so you know what fits before you load it.

Run several models at once. Load, unload, and hot-swap from the tray popover or with the in-chat command bar (⌘K on macOS).

07

The chat agent

The built-in chat does more than forward messages. It reads each task and routes it to the local model that fits: general, code, reasoning, or vision. It hot-swaps experts to fit available memory, keeping a small general model warm so the first token stays fast.

Pin any model yourself mid-conversation with ⌘K (macOS) or the command bar on other platforms. Long sessions stay sharp via observation masking: old tool output is trimmed before it reaches the model, so context stays useful rather than bloated.

08

Tools and privacy

The tool loop is sandboxed and local-first. Every tool is tagged LOCAL or ONLINE in the privacy panel:

Tool What it does Privacy
read_file Reads files from folders you've explicitly allowed LOCAL
web_fetch Fetches a URL, private-network access blocked ONLINE
web_search Searches via your own SearXNG instance (off until you set a URL) ONLINE
LOCAL ONLINE

The privacy panel shows every active tool's tag during a session. One Offline switch cuts all tool internet access at once. Flip it and the agent can still read files, but won't touch the network.

MCP (external tool servers): Coming soon Connect external MCP tool servers to the local agent. Not yet available.

09

Web search

SearXNG is a private, self-hosted metasearch engine. LlamaRanch uses it so the agent can search the web from one local endpoint, with no API key and nothing routed through a third-party service. It is an ONLINE tool, so the searches it runs do reach the internet, but there is no account, no tracking, and no middleman holding your queries.

One-command setup

Web search needs a container runtime, either Docker or Podman, installed and running. With that in place, one command sets everything up:

shell
$ npx @llamaranch/wizard websearch

That writes a local, loopback-only SearXNG (bound to 127.0.0.1:8888), enables the JSON format, turns the limiter off, writes a secret key, verifies the instance answers, and points LlamaRanch at it. The wizard's main setup offers this same step. Once it finishes, turn web search on in the chat and ask the agent something current.

It just runs

LlamaRanch starts the SearXNG container when the app launches and stops it when you quit. There is nothing to babysit.

Advanced

The config lives in ~/.llamaranch/searxng/ as a docker-compose.yml and config/settings.yml, and both are yours to edit. If you already run a SearXNG instance, point at it instead under Settings · Tools · SearXNG URL, and LlamaRanch will use that rather than managing its own.

Gotchas

  • You need Docker or Podman installed and running. Without a container runtime there is nothing to start.
  • If you hand-roll SearXNG for the API yourself, enable the json format and usually disable the limiter. The wizard does both for you.
  • Do not point it at a public SearXNG instance. JSON is normally disabled on those, and querying them this way is against their terms.
10

Configuration

The config file lives at:

macOS ~/Library/Application Support/llamaranch/config.json
Linux ~/.config/llamaranch/config.json
Windows %APPDATA%\llamaranch\config.json

Most keys are editable in the Settings window. Editing the file directly also works; changes take effect on restart.

Key Default Description
port 2276 Port the endpoint listens on
models_dir auto Directory where .gguf model files are stored
server_bin auto Path to the llama-server binary (auto-detected from PATH)
expose_to_network false Bind 0.0.0.0 instead of 127.0.0.1 (for other devices on your LAN)
models_max 3 Maximum number of models loaded concurrently
sleep_idle_seconds 600 Seconds of inactivity before an idle model unloads
general_model auto Model kept warm for general tasks and fast first tokens
embedding_model none Model used for /v1/embeddings requests
allowed_dirs [] Folders the read_file tool may access (local, explicit list)
searxng_url none URL of your SearXNG instance; enables web_search when set
offline_mode false Cut all tool internet access with one switch
hf_token none Hugging Face token for downloading gated models
11

Wizard reference

The wizard is published on npm as @llamaranch/wizard. Commands:

Command What it does
npx @llamaranch/wizard Full setup: hardware detection, llama.cpp install, model selection, app install
npx @llamaranch/wizard serve Start the endpoint router headlessly in the terminal (no app required)
npx @llamaranch/wizard update Pull and install the latest LlamaRanch release for your OS
npx @llamaranch/wizard --help Show usage and all options
npx @llamaranch/wizard --version Print the wizard version

The wizard calls the GitHub API to check for releases. If you run it frequently in CI or a script, set GITHUB_TOKEN in your environment to raise the API rate limit.

shell
# raise API rate limit
$ GITHUB_TOKEN=ghp_... npx @llamaranch/wizard update
12

Build from source

You need Rust, Node, and a llama-server binary on your PATH. The Tauri prerequisites page lists the current minimums for your OS.

shell 
$ git clone https://github.com/madalintat/LlamaRanch
$ cd LlamaRanch
$ npm install
$ npm run tauri dev      # hot-reload dev loop
$ npm run tauri build    # production build
